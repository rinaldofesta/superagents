## Test Structure

```{{language}}
describe('Module/Component', () => {
  // Setup
  beforeEach(() => { /* setup */ });
  afterEach(() => { /* cleanup */ });

  describe('feature', () => {
    it('should do X when Y', () => {
      // Arrange
      const input = createTestData();

      // Act
      const result = functionUnderTest(input);

      // Assert
      expect(result).toBe(expected);
    });

    it('should handle edge case', () => { ... });
    it('should throw on invalid input', () => { ... });
  });
});
```

## Test Naming

```
// Pattern: should [expected behavior] when [condition]
'should return empty array when no items match'
'should throw ValidationError when email is invalid'
'should update user when valid data provided'
```

## Mocking Guidelines

```{{language}}
// Mock external dependencies
vi.mock('./api', () => ({
  fetchData: vi.fn().mockResolvedValue({ data: [] })
}));

// Spy on methods
const spy = vi.spyOn(service, 'method');
expect(spy).toHaveBeenCalledWith(expected);

// Reset between tests
beforeEach(() => {
  vi.clearAllMocks();
});
```

## Rules

1. Test behavior, not implementation
2. One assertion per test (ideally)
3. Keep tests fast and isolated
4. Don't test external libraries
5. Use meaningful test data
